# Options

# index integer 0

开始幻灯片索引。0是第一张幻灯片。必须为整数，而不是字符串。

# getThumbBoundsFn function undefined

函数应返回一个对象，该对象的坐标将从其初始放大动画开始（或缩小动画结束）开始。

对象应包含三个属性：（x相对于文档的 X 位置），y（相对于文档的 Y 位置），w（元素的宽度）。高度将根据大图像的尺寸自动计算。例如，如果您返回{x:0,y:0,w:50}缩放动画，则动画将从页面的左上角开始。

函数有一个参数- index正在打开或关闭的项目。

在非模式模式下，应从x和中减去模板相对于视口的位置y。查看常见问题以获取更多信息。

计算缩略图位置的示例：

getThumbBoundsFn: function(index) {

    // find thumbnail element
    var thumbnail = document.querySelectorAll('.my-gallery-thumbnails')[index];

    // get window scroll Y
    var pageYScroll = window.pageYOffset || document.documentElement.scrollTop;
    // optionally get horizontal scroll

    // get position of element relative to viewport
    var rect = thumbnail.getBoundingClientRect();

    // w = width
    return {x:rect.left, y:rect.top + pageYScroll, w:rect.width};


    // Good guide on how to get element coordinates:
    // http://javascript.info/tutorial/coordinates
}

如果小缩略图的尺寸与大图像的尺寸不匹配，请考虑启用缩放+淡入淡出过渡。您可以通过添加选项来做到这一点showHideOpacity:true（尝试将其添加到上述 CodePen 中以测试其外观）。或通过添加完全禁用过渡hideAnimationDuration:0, showAnimationDuration:0。有关此的更多信息，请参见 FAQ。

如果要在动画使用过程中“隐藏”小缩略图opacity:0，请不要visibility:hidden或display:none。请勿在动画开始时强行绘制和布局，以免出现延迟。

# showHideOpacity boolean false

如果设置为，则false背景opacity和图像scale将被动画化（图像不透明度始终为 1）。如果设置为true：根 PhotoSwipe 元素opacity和图像scale将被动画化。

要仅启用opacity过渡（不带scale），请不要定义getThumbBoundsFnoption。

# showAnimationDuration integer 333

初始放大过渡持续时间（以毫秒为单位）。设置0为禁用。除了此 JS 选项之外，您还需要在 PhotoSwipe CSS 文件中更改过渡时间：

.pswp--animate_opacity,
.pswp__bg,
.pswp__caption,
.pswp__top-bar,
.pswp--has_mouse .pswp__button--arrow--left,
.pswp--has_mouse .pswp__button--arrow--right {
  -webkit-transition: opacity 333ms cubic-bezier(0.4, 0, 0.22, 1);
  transition: opacity 333ms cubic-bezier(0.4, 0, 0.22, 1);
}

如果使用的是 Sass，则只需更改_main-settings.scss 中的过渡持续时间变量。

# hideAnimationDuration integer 333

与上一个选项相同，仅用于关闭（缩小）过渡。打开 PhotoSwipe 后，会将pswp--open类添加到根元素，您可以使用它在 CSS 中应用不同的过渡持续时间。

# bgOpacity number 1

背景（.pswp__bg）不透明度。应为 0 到 1 之间的数字，例如0.7。此样式是通过 JS 而不是 CSS 定义的，因为此值用于一些基于手势的过渡。

# spacing number 0.12

幻灯片之间的间距比。例如，0.12将渲染为滑动视口宽度的 12％（四舍五入）。

# allowPanToNext boolean true

当当前项目放大时，允许滑动到下一个/上一个项目。选件始终false在不支持硬件触摸的设备上。

# maxSpreadZoom number 2

执行展开（缩放）手势时的最大缩放级别。2表示可以将图像放大到原始尺寸的 2 倍。此处尽量避免使用太大的值，因为太大的图像可能会导致移动设备上的内存问题（尤其是在 iOS 上）。

# getDoubleTapZoom function

函数应返回缩放级别，在双击手势后，用户单击缩放图标或鼠标单击图像本身时，图像将缩放到该级别。如果返回，1图像将被缩放到其原始大小。

默认值：

getDoubleTapZoom: function(isMouseClick, item) {

    // isMouseClick          - true if mouse, false if double-tap
    // item                  - slide object that is zoomed, usually current
    // item.initialZoomLevel - initial scale ratio of image
    //                         e.g. if viewport is 700px and image is 1400px,
    //                              initialZoomLevel will be 0.5

    if(isMouseClick) {

        // is mouse click on image or zoom icon

        // zoom to original
        return 1;

        // e.g. for 1400px image:
        // 0.5 - zooms to 700px
        // 2   - zooms to 2800px

    } else {

        // is double-tap

        // zoom to original if initial zoom is less than 0.7x,
        // otherwise to 1.5x, to make sure that double-tap gesture always zooms image
        return item.initialZoomLevel < 0.7 ? 1 : 1.5;
    }
}

每次启动放大动画时都会调用该函数。因此，随时可以根据其大小或屏幕 DPI 为不同的图像返回不同的值。

# loop boolean true

使用滑动手势时循环滑动。如果设置为true，则可以从最后一张图像滑动到第一张图像。false少于 3 张幻灯片时，总是选择。

此选项与箭头导航无关。箭头循环会永久打开。您可以通过制作自定义 UI 来修改此行为。

# pinchToClose boolean true

捏以关闭画廊手势。用户放大时，画廊的背景将逐渐淡出。手势完成后，图库将关闭。

# closeOnScroll boolean true

在页面滚动上关闭图库。选项仅适用于不支持硬件触摸的设备。

# closeOnVerticalDrag boolean true

垂直拖动和未缩放图像时，请关闭图库。总是false在使用鼠标时。

# mouseUsed boolean false

Option 允许您预定义是否使用鼠标。某些 PhotoSwipe 功能依赖于此功能，例如，仅在使用鼠标后才会显示默认的 UI 左/右箭头。如果设置为false，则 PhotoSwipe 将开始检测自身是否使用mouseUsed鼠标，并在找到鼠标时触发事件。

# escKey boolean true

esc键盘键以关闭 PhotoSwipe。可以动态更改选项（yourPhotoSwipeInstance.options.escKey = false;）。

# arrowKeys boolean true

键盘向左或向右箭头键导航。可以动态更改选项（yourPhotoSwipeInstance.options.arrowKeys = false;）。

# history boolean true

如果设置为false禁用历史记录模块（返回按钮关闭图库，每张幻灯片的唯一 URL）。您也可以history.js从构建中排除模块。

# galleryUID integer 1

画廊的唯一 ID。形成 URL 时由“历史记录”模块使用。例如，UID 1 画廊的第二张照片将有网址：http://example.com/#&gid=1&pid=2。

# galleryPIDs boolean false

为形成 URL 时使用的每个幻灯片对象启用自定义 ID。如果将选项设置为true，则幻灯片对象必须具有pid（图片标识符）属性，该属性可以是字符串或整数。例如：

var slides = [
    {
        src: 'path/to/1.jpg',
        w:500,
        h:400,
        pid: 'image-one'
    },
    {
        src: 'path/to/2.jpg',
        w:300,
        h:700,
        pid: 'image-two'
    },

    ...
];

...第二张幻灯片将具有 URL http://example.com/#&gid=1&pid=image-two。

在FAQ 部分中了解有关如何实现自定义 PID 的更多信息。

# errorMsg string

未加载图像时的错误消息。%url%将被替换为图片的网址。

默认值：

<div class="pswp__error-msg">
  <a href="%url%" target="_blank">The image</a> could not be loaded.
</div>

# preload array [1,1]

根据移动方向延迟加载附近的幻灯片。应为包含两个整数的数组，第一个-当前图像之前要预加载的项目数，第二个-当前图像之后要预加载的项目数。例如，如果将其设置为[1,3]，它将在当前图像之前加载 1 张图像，在当前图像之后加载 3 张图像。值不能小于 1。

# mainClass string undefined

具有类名称的字符串，该名称将添加到 PhotoSwipe（.pswp）的根元素中。可以包含多个由空格分隔的类。

# getNumItemsFn function

该函数应返回画廊中的物品总数。默认情况下，它返回幻灯片数组的长度。不要在这里放置非常复杂的代码，函数经常执行。

# focus boolean true

打开后，将焦点放在 PhotoSwipe 元素上。

# isClickableElement function

默认值：

isClickableElement: function(el) {
    return el.tagName === 'A';
}

函数应检查元素（el）是否可单击。如果是– PhotoSwipe 将不会调用preventDefault，click事件将会通过。功能应尽可能轻巧，因为它在拖动开始和拖动释放时执行了多次。

# modal boolean true

控制 PhotoSwipe 是否应扩展以占据整个视口。如果为 false，则 PhotoSwipe 元素将采用模板所定位父对象的大小。

# 默认 UI 选项

被加到以同样的方式和相同的对象为核心的选项。

// Size of top & bottom bars in pixels,
// "bottom" parameter can be 'auto' (will calculate height of caption)
// option applies only when mouse is used,
// or width of screen is more than 1200px
//
// (Also refer to `parseVerticalMargin` event)
barsSize: {top:44, bottom:'auto'},

// Adds class pswp__ui--idle to pswp__ui element when mouse isn't moving for 4000ms
timeToIdle: 4000,

// Same as above, but this timer applies when mouse leaves the window
timeToIdleOutside: 1000,

// Delay until loading indicator is displayed
loadingIndicatorDelay: 1000,

// Function builds caption markup
addCaptionHTMLFn: function(item, captionEl, isFake) {
    // item      - slide object
    // captionEl - caption DOM element
    // isFake    - true when content is added to fake caption container
    //             (used to get size of next or previous caption)

    if(!item.title) {
        captionEl.children[0].innerHTML = '';
        return false;
    }
    captionEl.children[0].innerHTML = item.title;
    return true;
},

    // `pswp` is the gallery instance object,

    return shareButtonOut;
}

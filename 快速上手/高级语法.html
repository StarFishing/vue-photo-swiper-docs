# 高级语法

如果你想要实现演示图那样，你只需要额外添加一些参数和配置项就可以了。我们提供了丰富的配置型，但是也需要您遵循我们的一些约定，如下面的示例

# 模板文件

这里我们将会增加两个重要的配置参数 gallerySelector 以及 gallerySelectorItem，这两个人参数是成对存在的，为了保证预览的实现，请您按照示例的模板结构进行安排您的布局，正如示例中 gallery 以及 gallery-item 的定义，您可以自己定义类名，但务必保证结构嵌套的规范。

<template>
  <div class="wrapper">
     <!--这个容器是十分必要的，插件内部会根据你提供的类名进行遍历子元素，从而拿到所有元素的src，以及
          其他的一些额外属性，这样做的目的也是避免您传入更多的参数,并且您也可以有多个这样的容器 -->
       <div class="gallery">
   <!-- 通常在这个容器上会放一些图片的其他信息，比如中号的图片和尺寸，目的是让小屏幕设备也能很好的展示，
        而不是原始的大图，因为图片太大会对过渡效果有一定影响，当然也可以不设置这个属性 -->
        <div class="gallery-item"
             v-for="(url, index) in images"
             :key="index"
             :data-origin="url.src" // 图片的原始地址，如果没有会去取img的src
             :data-size="url.w + 'x' + url.h" // 必须设置的原始图片的尺寸，否则无法进行任何过渡
             :data-med="url.src" // 如果有中号图片可以设置，我这里仅仅是作展示，也可以没有
             :data-med-size="url.w + 'x' + url.h" // 中号图片的尺寸
             @click="handlePreview(index)">
          <img :src="url.src"
               alt=""
               />
        </div>
      </div>
      <photo-swiper  @close="handleClose"
                    :visible="visible"
                    :options="options"
                    gallerySelector="gallery" // 指定一个容器
                    gallerySelectorItem="gallery-item" // 指定一个动画起始的元素
      ></photo-swiper>
    </div>
  </div>
</template>

# js 文件

这里我们增加了一些配置项

<script>
export default {
  data() {
    return {
      visible: false,
      images: [
        // 现在这个数据机构可以自己定义，只要保证参数按照要求传递到对应的属性上就可以了
        {
          src: 'lovely.png',
          w: 100,
          h: 100
        },
        {
          src: 'boy.png',
          w: 220,
          h: 400
        }
      ],
      options: {
        index: 0
      }
    }
  },
  methods: {
    handlePreview(index) {
      this.visible = true
      this.options.index = index // 这个配置可以让photo-swiper直接展示给定的这一页，否则从第一个开始
    },
    handleClose() {
      this.visible = false
    }
  }
}
</script>
